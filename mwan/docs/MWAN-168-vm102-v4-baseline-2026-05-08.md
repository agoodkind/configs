# MWAN-168 VM 102 v4 baseline + IPv6 BGP up (2026-05-08)

## Summary

VM 102 (opnsense-test2 on suburban) reached the `v4-baseline-2026-05-08`
snapshot with the testbed-shaped config imported and the IPv6 prefix
collision fixed. After re-attaching `net1` on `vmbr2`, BGP IPv4 came up
to both peers and BGP IPv6 came up to `3d06:bad:b01:201::3` with a v6
default route installed via FRR. The second v6 peer
(`3d06:bad:b01:201::4`, mwan-failover-test) cycles between
`Connect`/`Active`; v4 BGP to that peer is healthy, so the failure is
v6-specific reachability on the VM 950 side.

## Root cause: IPv6 prefix collision in substitutions

`mwan/testbed/opnsense/substitutions.yaml` carried two text-literal
rules that mapped distinct prod prefixes to the same testbed prefix:

```
v6 PRIVILEGED prefix shift: 3d06:bad:b01:1::  -> 3d06:bad:b01:201::
v6 WAN prefix shift:        3d06:bad:b01:fe:: -> 3d06:bad:b01:201::
```

The result: PRIVILEGED LAN (vlan0100) and WAN (vtnet1) both received
`3d06:bad:b01:201::/64`. FreeBSD installed the connected route on
vlan0100 (alphabetical priority over vtnet1), so packets to BGP peers
`3d06:bad:b01:201::3` and `3d06:bad:b01:201::4` egressed the wrong
interface and NDP failed.

## Fix

PRIVILEGED, GENERAL, and CAPTIVE shifted to the
`prod :N::/64 -> testbed :2N::/64` two-digit convention from
`MWAN-140-config-xml-transform-spec.md` section 10.1. WAN and
MANAGEMENT keep their existing testbed prefixes (`:201::/64` and
`:204::/64`) because they are the live testbed mwan-transit and SSH
paths. IPv6OnlyVLAN already followed the `2N` convention (`:264::/64`).

Substitution table changes (`mwan/testbed/opnsense/substitutions.yaml`):

- New `xpath_set`s targeting each interface's canonical `<ipaddrv6>`
  element so the per-interface address writes are explicit and
  collision-free (`PRIVILEGED LAN ipv6` -> `:21::1`, `GENERAL ipv6`
  -> `:22::1`, `IPv6OnlyVLAN ipv6` -> `:264::1`).
- `text_literal` `v6 PRIVILEGED prefix shift` retargeted to `:21::`.
- `text_literal` `v6 GENERAL prefix shift` retargeted to `:22::`.
- `text_literal` `v6 CAPTIVE prefix shift` retargeted to `:23::`.
- `text_literal` `v6 WAN prefix shift` left at `:201::` (live testbed
  mwan-transit prefix on suburban `vmbr2 3d06:bad:b01:201::5/64`).
- `text_literal` `v6 MANAGEMENT prefix shift` left at `:204::`
  (live testbed SSH path; tooling depends on it).

Verified order is collision-free: the post-substitution prefixes
(`:21::`, `:22::`, `:23::`, `:201::`, `:204::`, `:264::`, `:200::`,
`:2664::`) are pairwise non-substring of each other and non-substring
of the source prod literals (`:1::`, `:2::`, `:3::`, `:4::`, `:64::`,
`:fe::`, `::`, `:6464::`).
